# Техническое задание
## AGT — Project 911

**Версия:** 1.0  
**Дата:** 2026  
**Формат:** Web + PWA (мобильный сайт)

---

## 1. Назначение системы

AGT (Project 911) — платформа для проведения командных городских игр в реальном времени. Организатор создаёт игру с набором заданий и подсказок. Команды регистрируются, играют с мобильных устройств, вводят ответы и запрашивают подсказки. Система считает время и формирует итоговую таблицу.

---

## 2. Роли пользователей

| Роль | Описание |
|------|----------|
| **Суперадмин** | Владелец платформы. Управляет организаторами. |
| **Организатор** | Создаёт игры, задания, подсказки. Запускает и останавливает игру. Видит прогресс в реальном времени. |
| **Капитан команды** | Регистрирует команду, управляет составом, вводит ответы. |
| **Участник команды** | Видит задания и подсказки, но не может вводить ответы (опционально). |

---

## 3. Структура данных

### 3.1 Игра (Game)
- Название, описание
- Дата и время начала/окончания
- Статус: `draft` | `active` | `finished`
- Тип подсчёта очков (по времени / по количеству подсказок / смешанный)
- Максимальное число команд
- Пароль для входа команд (опционально)

### 3.2 Задание (Problem)
- Привязка к игре
- Порядковый номер (определяет очерёдность)
- Текст задания (поддержка Markdown + изображения)
- Список допустимых ответов (с нечётким сравнением: регистр, пробелы)
- Лимит времени на задание (опционально)
- Штраф за неверный ответ (в секундах или очках)

### 3.3 Подсказка (Tip)
- Привязка к заданию
- Тип: `1` (лёгкая) | `2` (средняя) | `3` (полная)
- Текст подсказки (Markdown + изображения)
- Задержка появления — через сколько секунд от начала задания становится доступна
- Штраф за использование (в секундах или очках)
- **Правило:** при взятии подсказки типа `3` автоматически засчитываются типы `1` и `2`

### 3.4 Команда (Team)
- Название, логотип (опционально)
- Привязка к игре
- Список участников (email/никнейм)
- Токен сессии (JWT)

### 3.5 Ответ (Answer)
- Команда, задание, введённый текст, время подачи
- Результат: `correct` | `wrong`

### 3.6 Использованная подсказка (UsedTip)
- Команда, подсказка, время взятия (в секундах от начала задания)

## 4. Функциональные требования

### 4.1 Панель организатора

- Создание/редактирование/удаление игр
- Конструктор заданий: текст, ответы, изображения, порядок drag-and-drop
- Конструктор подсказок для каждого задания
- Запуск и остановка игры одной кнопкой
- Дашборд в реальном времени:
  - Таблица команд с текущим заданием и временем
  - Лента событий (кто взял подсказку, кто ответил)
  - Ручной перевод команды на следующее задание (при необходимости)
- Экспорт результатов (CSV, PDF)

### 4.2 Интерфейс команды

- Авторизация по коду команды или ссылке-инвайту
- Страница текущего задания:
  - Текст задания
  - Таймер с начала задания
  - Доступные подсказки (с обратным отсчётом до появления)
  - Поле ввода ответа
  - Индикатор неверных попыток
- Уведомление о переходе на следующее задание (WebSocket / SSE)
- Итоговая таблица по завершении игры

### 4.3 Логика подсказок

1. Подсказка типа `1` доступна через N секунд от старта задания
2. Подсказка типа `2` доступна через M секунд (M > N)
3. Подсказка типа `3` доступна через K секунд (K > M)
4. При взятии подсказки `3` → в БД автоматически фиксируются `1` и `2` как использованные (если не взяты)
5. Нельзя взять подсказку раньше срока — проверяется на сервере, клиент лишь скрывает кнопку

### 4.4 Подсчёт результатов

- Основной критерий: время прохождения всех заданий
- Штрафное время добавляется за:
  - Каждую взятую подсказку
  - Каждый неверный ответ
- Итоговая таблица: место, команда, чистое время, штраф, итого

## 5. Требования к безопасности

### 5.1 Аутентификация и авторизация
- Организаторы: email + пароль, JWT access token (15 мин) + refresh token (7 дней) в httpOnly cookie
- Команды: уникальный токен команды, хранится в httpOnly cookie
- Все защищённые маршруты проверяют токен на сервере
- Разделение прав: команда не может получить данные другой команды

### 5.2 Защита от атак
- Все входные данные валидируются и санируются на сервере
- Параметризованные SQL-запросы (ORM) — никакой ручной конкатенации в запросах
- Rate limiting на ввод ответов (не более 1 попытки в 3 секунды на команду)
- Rate limiting на все публичные эндпоинты (100 запросов в минуту с одного IP)
- CSRF-защита для мутирующих запросов
- Helmet.js (или аналог) — заголовки безопасности (CSP, HSTS, X-Frame-Options и др.)
- Проверка доступности подсказки строго на сервере (клиент не доверяется)

### 5.3 Хранение данных
- Пароли: bcrypt, cost factor ≥ 12
- Секреты (JWT secret, DB credentials): только через переменные окружения, никогда в коде
- Файл `.env` в `.gitignore`
- БД не открыта в интернет — только localhost или приватная сеть

### 5.4 Служебные эндпоинты
- Просмотр таблиц БД — только для суперадмина с обязательной авторизацией
- Логи приложения не содержат паролей и токенов

## 6. Технический стек (рекомендуемый)

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| **Backend** | Node.js + TypeScript + Fastify | Быстрый, типизированный, хорошая экосистема |
| **ORM** | Prisma | Типобезопасные запросы, миграции, защита от SQL-инъекций |
| **БД** | PostgreSQL | Надёжная, поддержка транзакций |
| **Frontend** | React + Vite | PWA из коробки, быстрая разработка |
| **Реальное время** | SSE (Server-Sent Events) | Проще WebSocket для односторонних обновлений |
| **Стили** | Tailwind CSS | Быстрая вёрстка, мобильный first |
| **Аутентификация** | JWT + httpOnly cookies | Защита от XSS |
| **Деплой** | Docker + Nginx | Простой, воспроизводимый |

## 7. Дизайн-система (UI)

### 7.1 Общая концепция

AGT — Project 911 использует **тёмную тему** как основную и единственную. Интерфейс вдохновлён эстетикой ночного города: глубокие тёмные фоны, неоновые акценты, высокий контраст для читаемости в условиях плохого освещения.

### 7.2 Цветовая палитра

| Назначение | Цвет | HEX |
|------------|------|-----|
| Фон приложения | Почти чёрный | `#0d0f12` |
| Фон карточек / панелей | Тёмно-серый | `#16181c` |
| Фон элементов (инпуты, кнопки) | Тёмный | `#1e2126` |
| Граница / разделитель | Серый | `#2a2d35` |
| Основной текст | Светло-серый | `#ccd0d5` |
| Вторичный текст | Серый | `#6e6f72` |
| Акцент (основной) | Синий неон | `#4a9eca` |
| Акцент (успех / верный ответ) | Зелёный | `#92d22e` |
| Акцент (ошибка / штраф) | Красный | `#ca4a4a` |
| Акцент (предупреждение / подсказка) | Розовый | `#ca4a72` |
| Акцент (таймер / время) | Оранжевый | `#FF9900` |

### 7.3 Типографика

- Основной шрифт: **Inter** (или Roboto) — хорошая читаемость на тёмном фоне
- Моноширинный (таймеры, коды): **JetBrains Mono** или **Roboto Mono**
- Минимальный размер текста на мобильном: 16px
- Заголовки: полужирные, цвет акцента `#4a9eca`

### 7.4 Компоненты

**Кнопки:**
- Primary: фон `#4a9eca`, текст тёмный — для основных действий (отправить ответ, взять подсказку)
- Secondary: граница `#2a2d35`, текст `#ccd0d5` — для вторичных действий
- Danger: фон `#ca4a4a` — для деструктивных действий (удалить, остановить игру)
- Все кнопки: скруглённые углы 6px, полужирный текст

**Инпуты:**
- Фон `#1e2126`, граница `#2a2d35`
- При фокусе: граница `#4a9eca` с тонким свечением (box-shadow)
- Текст `#ccd0d5`, плейсхолдер `#6e6f72`

**Карточки:**
- Фон `#16181c`, граница `#2a2d35`
- Лёгкая тень: `0 2px 8px rgba(0,0,0,0.4)`

**Таймер:**
- Моноширинный шрифт, крупный размер
- Цвет: оранжевый `#FF9900` — привлекает внимание
- При критическом времени (< 60 сек): мигание или смена цвета на красный

**Подсказки:**
- Заблокированная: серый текст + замок-иконка + обратный отсчёт
- Доступная: акцент `#ca4a72` (розовый), пульсирующая анимация
- Использованная: приглушённый, зачёркнутый

**Статусы ответа:**
- Верно: зелёная вспышка + иконка ✓
- Неверно: красная вспышка + тряска инпута (CSS animation)

### 7.5 Логотип и брендинг

- Название проекта: **AGT** крупно + `Project 911` мелко под ним
- Стиль: технократичный, без засечек
- Использование красного `#ca4a4a` как второго акцентного цвета в логотипе (отсылка к «911»)

## 8. API (основные эндпоинты)

### Публичные
```
POST /auth/login            — вход организатора
POST /teams/join            — вход команды по коду
```

### Команда (требует токен команды)
```
GET  /game/current          — текущее состояние игры для команды
GET  /problem/current       — текущее задание
POST /problem/answer        — отправить ответ
GET  /tips                  — список подсказок с таймерами
POST /tips/:id/use          — взять подсказку
GET  /results               — итоговая таблица (после окончания)
```

### Организатор (требует JWT)
```
GET/POST/PUT/DELETE /games
GET/POST/PUT/DELETE /games/:id/problems
GET/POST/PUT/DELETE /problems/:id/tips
POST /games/:id/start
POST /games/:id/stop
GET  /games/:id/dashboard   — SSE-стрим с прогрессом
GET  /games/:id/results/export
```

## 9. Нефункциональные требования

- **Время ответа:** < 300 мс для игровых запросов
- **Мобильный интерфейс:** корректная работа на экранах от 320px, PWA (добавление на главный экран, работа при плохом интернете)
- **Браузеры:** последние 2 версии Chrome, Firefox, Safari
- **Доступность:** семантическая вёрстка, читаемые шрифты (≥ 16px для основного текста)
- **Логирование:** структурированные логи (JSON) с уровнями info/warn/error

## 10. Схема базы данных (упрощённая)

```
User (организатор)
  id, email, password_hash, role, created_at

Game
  id, title, description, status, scoring_type,
  wrong_answer_penalty, created_by (→ User), created_at

Problem
  id, game_id (→ Game), order_num, text, time_limit

ProblemAnswer (допустимые ответы)
  id, problem_id (→ Problem), answer_text

Tip
  id, problem_id (→ Problem), type (1|2|3),
  text, delay_seconds, penalty_seconds

Team
  id, game_id (→ Game), name, token_hash, created_at

TeamMember
  id, team_id (→ Team), nickname

AnswerLog
  id, team_id (→ Team), problem_id (→ Problem),
  submitted_text, is_correct, submitted_at

UsedTip
  id, team_id (→ Team), tip_id (→ Tip), used_at_seconds
```
